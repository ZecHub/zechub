## Resumen

- Acepta pagos usando una dirección blindada
- Evita exponer los datos de transacciones de los clientes
- Usa memos o sistemas off-chain para el seguimiento de pedidos
- Mantén privados los ingresos de tu negocio

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/warning-error-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icono de advertencia"/> El problema

Los pagos tradicionales con criptomonedas exponen:
- El comportamiento de compra del cliente
- Los ingresos del comerciante
- El historial de transacciones

Esto provoca:
- Pérdida de privacidad del cliente
- Filtraciones de inteligencia competitiva
- Transparencia financiera que quizá no deseas

<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-lock.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icono de caja de herramientas"/> ¿Por qué Zcash?

Zcash permite:
- Pagos privados de clientes
- Seguimiento confidencial de ingresos
- Protección de la actividad empresarial


<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-toolbox.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icono de caja de herramientas"/> Qué necesitas

- Una billetera de Zcash compatible con transacciones blindadas
- Un flujo de pago simple (manual o integrado)
- Opcional: sistema de seguimiento de pedidos


<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icono de paso"/> Guía paso a paso

### Paso 1: Configura una dirección de pago

- Genera una dirección blindada (z)
- Úsala para recibir pagos


### Paso 2: Muestra las instrucciones de pago

Para cada transacción:
- Muestra tu dirección z
- Indica el importe del pago
- Opcionalmente incluye una referencia en el memo

### Paso 3: Haz seguimiento de los pedidos

Usa:
- Memos (p. ej., ID del pedido)
- O seguimiento off-chain (base de datos, hoja de cálculo)


Ejemplo:
- Pedido #4821


### Paso 4: Confirma el pago

- Supervisa las transacciones blindadas entrantes
- Haz coincidir el memo o el importe con el pedido


### Paso 5: Mantén buenas prácticas de privacidad

- Evita reutilizar direcciones públicamente si es posible
- No expongas el historial de transacciones
- Usa billeteras separadas para distintas unidades de negocio si es necesario


<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-cancel.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icono de cancelar"/> Errores comunes que debes evitar

- Usar direcciones transparentes para pagos  
- Publicar pruebas de transacción públicamente  
- Vincular la identidad del cliente con datos de la blockchain  
- No hacer un seguimiento adecuado de los pagos
