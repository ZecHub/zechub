# Intercambio de archivos P2P con IPFS y el navegador Brave

Podemos usar IPFS para compartir archivos multimedia grandes entre nosotros de forma Peer-to-Peer.

Entonces, ¿cuál es el problema?: las puertas de enlace de IPFS no funcionan con archivos grandes, agotarán el tiempo de espera.

**Requisitos**: [IPFS Desktop](https://docs.ipfs.tech/install/ipfs-desktop/) + [navegador Brave](https://brave.com/)

tldr, el video está al final.

Podemos usar esto entre nosotros en el ZF AV Club para transferir archivos multimedia grandes y documentarlo para todos ustedes porque es divertido y súper genial.

*ADEMÁS*, colaboraremos directamente con el equipo de IPFS en Protocol Labs para desarrollar este caso de uso.
¡Otro beneficio que surgió de las conexiones hechas en dWeb Camp!

Sin IPFS instalado en la máquina receptora Y usando un navegador que no sea Brave: tendrás que esperar a que la puerta de enlace cargue el archivo. *Esto devolverá un error 504 con archivos grandes.*

A continuación: cómo fijar en tu máquina los archivos de otras personas para que no tengamos el problema de un único punto de fallo cuando el nodo original se desconecte.
