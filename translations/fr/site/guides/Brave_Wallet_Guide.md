# Comment envoyer et recevoir du Zcash (ZEC) protégé avec Brave Wallet : guide pas à pas pour la confidentialité

Brave Wallet est un portefeuille crypto sécurisé et natif, intégré directement au navigateur Brave - aucune extension requise.


## Pourquoi la prise en charge de ZEC protégé est importante


Zcash est l'une des seules cryptomonnaies à offrir la confidentialité par défaut grâce aux transactions protégées.

Brave Wallet prend désormais en charge ZEC protégé Orchard, permettant des transactions privées sans logiciel tiers.

Les utilisateurs peuvent maintenant envoyer/recevoir des ZEC sans révéler leurs soldes ni leurs métadonnées - le tout dans le navigateur Brave.

## Partie 1 : Configurer Brave Wallet

**Étapes couvertes :**

1. Ouvrez le navigateur Brave (version desktop uniquement pour le moment)

2. Cliquez sur l’icône Wallet ou allez à brave://wallet

3. Configurez votre portefeuille (créer ou importer)

4. Accédez au tableau de bord crypto


![img1](https://github.com/user-attachments/assets/f54cd1a1-8569-4925-ba1c-7597d030593e)


## Partie 2 : Ajouter Zcash (ZEC) à Brave Wallet

**Étapes couvertes :**

1. Dans le tableau de bord Wallet, cliquez sur "Manage Assets"

2. Recherchez ZEC et activez-le

3. ZEC apparaît maintenant dans votre portefeuille


![img2](https://github.com/user-attachments/assets/6f2b2190-cf55-4394-9d5f-29ff9b5bb525)


## Partie 3 : Recevoir des ZEC protégés (Unified Address)

**Étapes couvertes :**

1. Cliquez sur ZEC dans vos actifs

2. Sélectionnez "Receive"

3. Brave génère une Unified Address (UA) - une seule adresse qui prend en charge à la fois les transactions protégées et transparentes

4. Copiez ou scannez le code QR

5. Partagez votre UA avec l’expéditeur



![img3](https://github.com/user-attachments/assets/53c940b6-1a03-4fa7-aefa-d3478f678a88)



## Partie 4 : Envoyer des ZEC protégés en privé

**Étapes couvertes :**

1. Sélectionnez ZEC -> Send

2. Collez la Unified Address ou l’adresse protégée du destinataire

3. Saisissez le montant

4. Confirmez les détails de la transaction

5. Cliquez sur Send - les fonds sont maintenant transférés en privé, en utilisant le pool Orchard


![img4](https://raw.githubusercontent.com/Kellyjoe8/zechub/refs/heads/main/Internet_20250808_172118_4.webp)




## Types d’adresses

Voici une explication de chaque type d’adresse Zcash :

**t-addr (t1...)** - Les adresses transparentes fonctionnent comme les adresses Bitcoin, ce qui signifie que les transactions sont publiques sur la blockchain. Elles sont plus faciles à utiliser avec des portefeuilles non-Zcash, mais n’offrent pas de confidentialité.

**z-addr (zs...)** - Les adresses protégées utilisent la technologie de confidentialité de Zcash (Sprout/Sapling) pour masquer l’expéditeur, le destinataire et le montant. Seules les Viewing Key ou le propriétaire peuvent révéler les détails.

**UA (u... or QR)** - Les Unified Address combinent plusieurs types d’adresses (transparentes, protégées, Orchard) en une seule. Cela permet une compatibilité maximale tout en privilégiant la confidentialité lorsque c’est possible.
