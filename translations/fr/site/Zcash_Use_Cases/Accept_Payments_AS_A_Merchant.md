## TL;DR

- Accepter des paiements à l’aide d’une adresse blindée
- Éviter d’exposer les données de transaction des clients
- Utiliser des mémos ou des systèmes hors chaîne pour le suivi des commandes
- Préserver la confidentialité des revenus de l’entreprise

Les paiements crypto traditionnels exposent :
- Le comportement d’achat des clients
- Les revenus du commerçant
- L’historique des transactions

Cela entraîne :
- Une perte de confidentialité pour les clients
- Des fuites d’informations concurrentielles
- Une transparence financière que vous ne souhaitez peut-être pas

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icône étape"/> Guide étape par étape

### Étape 1 : Configurer une adresse de paiement

- Générer une adresse blindée (z)
- L’utiliser pour recevoir des paiements


### Étape 2 : Afficher les instructions de paiement

Pour chaque transaction :
- Afficher votre adresse z
- Indiquer le montant du paiement
- Inclure éventuellement une référence dans le mémo

### Étape 3 : Suivre les commandes

Utiliser :
- Les mémos (par ex., ID de commande)
- Ou un suivi hors chaîne (base de données, feuille de calcul)


Exemple :
- Commande #4821


### Étape 4 : Confirmer le paiement

- Surveiller les transactions blindées entrantes
- Faire correspondre le mémo ou le montant à la commande


### Étape 5 : Maintenir une bonne hygiène de confidentialité

- Éviter de réutiliser publiquement les adresses si possible
- Ne pas exposer l’historique des transactions
- Séparer les portefeuilles pour différentes unités de l’entreprise si nécessaire


<br/>

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/icons8-cancel.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icône annulation"/> Erreurs courantes à éviter

- Utiliser des adresses transparentes pour les paiements  
- Publier publiquement des preuves de transaction  
- Lier l’identité du client aux données de la blockchain  
- Ne pas suivre correctement les paiements
