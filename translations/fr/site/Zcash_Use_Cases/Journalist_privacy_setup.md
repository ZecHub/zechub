## TL;DR

- Utilisez des adresses blindées pour toutes les transactions
- N’exposez jamais publiquement l’identité du portefeuille
- Protégez vos sources en évitant les paiements traçables
- Utilisez les mémos avec précaution pour une communication sécurisée

## <img src="https://raw.githubusercontent.com/amochuko/zechub/82d2046091b73a626d818571a978fcaffdc7ebf4/assets/icons/ladder-svgrepo-com.svg" width="24" height="24" className="inline-block align-middle mr-1 p-[2px]" alt="icône d’étape"/> Guide étape par étape


### Étape 1 : Créer un portefeuille dédié

- Utilisez un portefeuille réservé au travail sensible
- Ne le mélangez pas avec des transactions personnelles

### Étape 2 : Utiliser uniquement des adresses blindées

- Effectuez toujours vos transactions avec des z-addresses
- Évitez entièrement les transactions transparentes


### Étape 3 : Protéger les paiements aux sources

Si vous payez des sources :
- Utilisez des transactions blindées (z → z)
- Évitez les schémas temporels prévisibles
- Ne réutilisez pas les adresses


### Étape 4 : Faire attention aux mémos

- N’incluez PAS d’informations sensibles dans les mémos
- Utilisez des références neutres ou codées si nécessaire


### Étape 5 : Maintenir la sécurité opérationnelle

- Évitez de lier le portefeuille à votre identité
- Ne confirmez pas publiquement les transactions
- Utilisez des canaux séparés pour la communication
