# Step-by-Step Guide: Submitting a Grant Application to Zcash Community Grants (ZCG)

_As of January 2026, Zcash Community Grants (ZCG) uses GitHub for formal grant submissions, with the Zcash Community Forum serving as the venue for public discussion and feedback. This process replaces the former Submittable platform._

Applicants must comply with ZCG terms, including conflict-of-interest disclosure, code of conduct adherence, and transparency requirements.

## 2. Prepare Your Grant Application Content

ZCG applications are submitted using a structured GitHub issue template. Prepare the following information in advance:

### Required Information
- Agreement to ZCG terms (including KYC for large grants and milestone validation)
- Application owner(s) (GitHub usernames)
- Organization or individual name
- How you learned about ZCG
- Requested funding amount (USD)
- Grant category (e.g., Wallets, Community, Infrastructure)

### Team & Project Details
- Project lead
- Team members (names, roles, experience, responsibilities)
- Project summary (short, high-level overview)
- Problem statement
- Proposed solution
- Technical approach (if applicable)
- Dependencies and integrations
- Risks and mitigation strategies
- Success metrics
- Potential side effects or trade-offs

### Budget & Funding
- Detailed budget breakdown:
  - Compensation
  - Software/services
  - Hardware
  - Other costs
- Total requested amount
- Previous ZCG or other funding (if any)
- Other funding sources
- Startup funding justification (if applicable)

### Milestones
- Milestone descriptions
- Deliverables
- Target dates
- Payment amounts per milestone

### Supporting Material
- Documentation links
- Design mockups
- GitHub repositories
- Research or technical specs

> **Note:**  
> Projects involving code are expected to follow open-source best practices. A `CONTRIBUTING.md` file and upstream contribution plans may be required.

## 3. Submit the Application via GitHub

1. Visit the ZCG GitHub repository:  
   https://github.com/ZcashCommunityGrants/zcashcommunitygrants

2. Navigate to **Issues â†’ New Issue**

3. Select the **Grant Application** template  
   (or use the direct link):  
   https://github.com/ZcashCommunityGrants/zcashcommunitygrants/issues/new?template=grant_application.yaml

4. Complete the YAML-based issue form using your prepared information.

5. Submit the issue.

Once submitted:
- The issue will automatically receive labels such as:
  - `Pending Grant Application`
  - `ðŸ“‹ Grant Application`

For submission-related questions, contact:  
**grants@fpriv.org**

## 4. Create a Forum Post for Community Feedback (Required)

Immediately after submitting your GitHub issue:

1. Go to the Zcash Community Forum Grants category:  
   https://forum.zcashcommunity.com/c/grants/33

2. Create a new topic titled:  
   **Grant Application Discussion: _Your Project Name_**

3. Include:
   - A concise summary of your proposal
   - Key objectives and deliverables
   - A direct link to your GitHub grant application issue

4. Monitor and actively respond to community feedback.

> Community discussion is a required part of the ZCG process and directly informs committee deliberation and voting.

---

## 5. Review, Due Diligence, and Voting

After validation:
- Your GitHub issue will be labeled **ðŸ‘€ Ready For ZCG Review**
- ZCG reviews applications on a **bi-weekly cadence**
- The committee considers:
  - Technical merit
  - Budget clarity
  - Alignment with Zcash goals
  - Community feedback

ZCG may request:
- Clarifications
- Technical reviews
- Budget or milestone adjustments

### Decision Outcomes
- **Approved**:  
  You will be contacted for KYC and payment setup through the Zcash Foundation.
- **Rejected**:  
  Feedback and reasoning will be provided publicly on GitHub and the forum.

---

## 6. Post-Approval Responsibilities

If your grant is approved:

- Post regular progress updates on:
  - The GitHub issue
  - The linked forum thread
- Submit milestone deliverables for review before payouts
- Follow ZCG instructions for milestone payment requests
- Maintain open-source standards and documentation
- Acknowledge ZCG support publicly when appropriate
- Promote milestones via social media (tagging @ZcashFoundation and @ZcashCommGrants)
